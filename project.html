About the project

Why do we make a repository?

A github repository, also known as "repo" is a place where you can store your projects and all of your code/files, along with the entire history of the project progress.

Having these things all in one place where both you and your team can access it, revert changes, and view changes can help alot especially for live environments in order to prevent conflicts and crashes.


Steps to make a repository

    • Step one: Sign in/sign up to Github.
      Step two: Click the "+" icon on the top right and select "New Repository".
      Step three: Fill in the required details e.g name, description and visibility
      Step four: Clone the Repository to your computer, by opening the terminal and typing "git clone https://github.com/your-username/your-repo-name.git
      Step five: Move into your repo folder using "cd your-repo-name".
      Step six: Add the documents you wish to add using "touch file-name".
      Step seven: Commit and Push changes made to update your repo.
    • If you wish to work together on a private project, and contribute as a team, you have to make the person who is wanting to work on it a "collaborator"
    • If you wish to contribute to a public project you have to use a "fork"
      Step one: Open the repository page.
      Step two: Click the "fork" button that is located at the top right. Github will then create a repository in your account.
      Step three: In your VScode terminal, use the "git clone" command with the link to that repository.
      Step four: Change your directory to the newly cloned repository.
      Step five: Set the original repository that you cloned from as the "upstream" using the command "git remote add upstream" command with the link to the original repository.
